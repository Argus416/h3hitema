Objectif de la journée, deployer 2 services applicatifs avec python et docker. 
Pour cela vous pouvez suivre la to do list suivante (pas nécessairement dans l’ordre)

✅ 01- Finir de développer son application streamlit.io
✅ 02- Préparer votre deploiement streamlit en créant un fichier requirement.txt contenant les packages dont votre app a besoin (se renseigner sur ce fichier, qui est un standard de dev)
✅ 03- Regardez comment travailler dans un environement virtuel et les differentes librairies qui le permettent (virtualenv, pyenv…)
 04- Déployer son app avec streamlit et m’envoyer le lien par GitHub 
✅ 05- Faire une veille technologique sur l’outil docker 
✅ 06- Faire une « cheat sheet » avec les principales commandes docker que vous avez compris 
07- Finir de developper son app template FastAPI (aller jusqu’au chapitre 7)
✅ 08- Faire tourner votre application FastAPI dans un environnement virtuelle 
09- Trouver le moyen de générer les dépendances (le fichier requirements.txt) automatiquement a partir de la configuration de votre environement 
✅ 10- Mettre dans un docker container ces deux projet (streamlit & FastAPI)

11- (Bonus) faire des test de votre application FastAPI (regarder la lib TestClient)
12- (Bonus) Monter une base de données Postgres via docker qui communique avec votre API 
13- (Bonus) se renseigner sur les volumes docker et monter un volume pour votre application
